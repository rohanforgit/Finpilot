# 📘 FinPilot Product Experience
Version: 1.0

---

# Product Experience Philosophy

FinPilot should feel calm.

Managing money is stressful enough.

The application should never overwhelm users with charts, numbers, complicated terminology or financial jargon.

Instead, every screen should answer one simple question.

The user should never wonder

"What do I do now?"

The application should always guide them.

---

# Experience Goals

The application should make users feel

✓ Organized

✓ Confident

✓ In Control

✓ Relaxed

Never

❌ Guilty

❌ Confused

❌ Overwhelmed

# Home Experience

When the user opens the application,

they should immediately know

How much money remains

How the month is progressing

Whether everything is okay

The Home page is called

Financial Snapshot

The first screen should contain

Remaining Money

Plan Progress

One AI Insight

Savings Bucket Preview

Recent Expenses

Quick Upload Button

Recommendation Preview

Nothing else.

No clutter.

---

# Monthly Plan Experience

Sections

Income

Fixed Expenses

Investments

Annual Savings

Miscellaneous Allowance

Remaining Money

# Expense Upload Experience

The upload process should feel magical.

User selects

Paytm

PhonePe

Google Pay screenshots

AI immediately starts processing.

Loading animation should communicate

Reading...

Understanding...

Categorizing...

Preparing...

Not simply

Loading...

---

# OCR Review Experience

Nothing is saved automatically.

Every transaction appears inside a review table.

User can

Edit

Delete

Change category

Mark as Planned

Mark as Miscellaneous

Approve everything with one click.

---

# AI Experience

AI should never feel like ChatGPT.

AI should feel like a financial assistant working quietly in the background.

Recommendations appear naturally.

No chatbot page.

No long conversations.

Small insights.

Short recommendations.

Helpful observations.

---
